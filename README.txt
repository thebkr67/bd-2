Telegram бот: автоответ "принято" на "+" в комментариях канала

Что было исправлено:
- добавлена проверка токена до запуска
- теперь бот не падает с непонятной ошибкой InvalidToken, если токен не вставлен
- можно передавать токен через переменную окружения BOT_TOKEN

Как запустить:

1. Создайте бота через @BotFather
2. Возьмите токен вида:
   123456789:AAExampleToken_xxxxxxxxxxxxxxxxxxx

Вариант 1. Вписать токен в файл main.py:
   BOT_TOKEN = "ВАШ_ТОКЕН"

Вариант 2. Передать токен через переменную окружения:
   Windows PowerShell:
   $env:BOT_TOKEN="ВАШ_ТОКЕН"
   python main.py

   Linux/macOS:
   export BOT_TOKEN="ВАШ_ТОКЕН"
   python main.py

3. Установите зависимости:
   pip install -r requirements.txt

4. В @BotFather отключите privacy mode:
   /setprivacy -> Disable

5. Добавьте бота в группу обсуждения канала
   Лучше выдать права администратора

6. Запустите:
   python main.py

Как работает:
- бот отвечает только на сообщение "+"
- бот отвечает только если это комментарий к посту канала
- в обычной переписке группы не отвечает

Если бот не реагирует:
- проверьте, что бот добавлен именно в группу обсуждения, а не только в канал
- проверьте, что privacy mode отключен
- проверьте, что комментарий написан как ответ под постом канала
